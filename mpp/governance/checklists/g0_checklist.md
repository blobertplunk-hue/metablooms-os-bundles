# G0 — Context Brief Reviewer Checklist

Complete this before writing the G0 receipt. Check each item honestly.

## Completeness

- [ ] The real user story names a specific type of person (not "the user")
- [ ] The real user story describes a specific action (not "uses the feature")
- [ ] The Definition of Done is concrete enough that two engineers would independently agree on it
- [ ] Hidden constraints are listed (or explicitly stated as none with justification)
- [ ] Failure cost is assessed (what breaks, for whom, how fast)
- [ ] Prior attempts are documented (or explicitly stated as first attempt)

## Quality

- [ ] I can read this brief and know what "done" looks like without asking questions
- [ ] The Definition of Done could be turned into a test assertion right now
- [ ] There is nothing in this brief that is vague enough to be interpreted two ways
- [ ] The brief reveals at least one constraint the research operator might not have known

## Verdict

- [ ] **quality_verdict:** SUFFICIENT — the brief meets all criteria above
- [ ] **quality_verdict:** INSUFFICIENT — at least one item above is unchecked; list gaps
